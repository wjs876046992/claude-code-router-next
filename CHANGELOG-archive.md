## English

| Version | Release Details |

| **v2.3.11** | <ul><li>**Fix First Request of a New Session Bypassing Project Routing (Session Detection Race)**: The first request of a new session (e.g. the title-generation meta request, which arrives ~tens of ms before the main request) could land before the session transcript file is flushed to disk, so project matching failed and it fell back to the global `Router` — bypassing project-level routing (e.g. a project with `enableFamilyRouting` off still went through global family routing). On a cache miss the lookup now retries briefly (up to 3×50ms) to give the file time to appear, and retries only once per session so un-managed sessions don't pay the delay on every request.</li></ul> |
| **v2.3.10** | <ul><li>**Fix Title-Generation Requests Triggering `think` Routing**: `thinking: {type: "disabled"}` is a truthy object and was previously misread as "thinking enabled", routing the request to the `think` model; now only `thinking.type === "enabled"` enters the `think` scenario, so projects with family routing disabled no longer get routed to the global think model.</li><li>**Fix Empty Model When Default Is Unhealthy With No Fallback**: When `Router.default` is unhealthy (fail-pool) and no fallback is available, it previously returned an empty model, causing a synthetic "provider not found" error; it now retries the configured default model with the health check skipped as a last resort, so Claude Code gets a real upstream response to retry on.</li></ul> |
| **v2.3.9** | <ul><li>**Codex Managed Account Token Auto-Refresh**: Adds a background scheduler (first run 60s after startup, then every 30 minutes) that checks every managed Codex account — active or not — and proactively refreshes it once its `access_token` is within 24 hours of expiry or it hasn't been refreshed in 7 days, exchanging the stored `refresh_token` and writing the result back to the account store; if the account is currently active, `~/.codex/auth.json` is backed up and synced as well. Can be disabled via `Clients.codex.autoRefreshTokens` (enabled by default).</li><li>**Fix Runtime Fallback Retry Ignoring Project-Level `enableFallback`**: When an in-flight request failed (e.g. rate limit) and triggered a retry fallback, it previously read the global `Router.enableFallback` and global top-level `fallback` config directly, ignoring a project's `enableFallback: false` and custom `Router.fallback`.
